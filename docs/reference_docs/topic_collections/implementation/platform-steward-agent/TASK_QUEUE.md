## Queue rules

1. Allowed statuses are only: `READY`, `IN_PROGRESS`, `PENDING`, `BLOCKED`, `DONE`.
2. Only one task may be `IN_PROGRESS` globally unless a lane rule explicitly says otherwise.
3. Promote `PENDING` to `READY` only when dependencies are `DONE`.
4. Deterministic pick order is `P0` before `P1`, then lowest task ID.
5. Every row must run its `Verify` commands before moving to `DONE`.
6. The org-owned One-of-One Operator clone remains the only default authenticated `desktop` route.
7. Mother actions must never silently mutate customer-owned clones.
8. Authenticated customer access to Mother must be explicit; guest onboarding entrypoints may continue to route into Mother onboarding mode.
9. Mother implementation must reuse the existing template lifecycle and distribution machinery; no parallel publish or sync engine.
10. Quinn must be treated as the current implementation base and compatibility alias during rollout, not as a separate peer to Mother.
11. Do not revert or bulldoze dirty in-flight work in telephony, Twilio, templates, layers, routing, agent UI, chat-side `configure_agent_fields`, or default-operator bootstrap.
12. Sync `INDEX.md`, `MASTER_PLAN.md`, `TASK_QUEUE.md`, and `SESSION_PROMPTS.md` at lane milestones.
