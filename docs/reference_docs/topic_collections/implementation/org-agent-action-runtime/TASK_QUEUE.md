**Source request:** Create a queue-first implementation plan for a full org-owned agent activity/action runtime that routes Veronica-style telephony through the existing One-of-One Operator runtime, keeps platform-owned CRM state canonical, separates immutable activity from actionable pending work, and lands a strong internal-only V1 before broad external CRM execution.

---

## Queue rules

1. Allowed statuses are only: `READY`, `IN_PROGRESS`, `PENDING`, `BLOCKED`, `DONE`.
2. Only one row may be `IN_PROGRESS` globally unless an explicit lane exception is added here first.
3. Promote `PENDING` to `READY` only when every dependency is `DONE`.
4. Deterministic pick order is `P0` before `P1`, then lowest task ID.
5. Every row must run its `Verify` commands before moving to `DONE`.
6. Platform-owned CRM objects, links, activities, action items, and `objectActions` remain canonical; external CRM writes are downstream projections only.
7. Immutable activity and mutable pending work must stay separate: `org_agent_activity` never reopens, `org_agent_action_item` carries owner workflow state.
8. Veronica-style telephony must enter through the existing One-of-One Operator runtime and `agentSessions` flow; no standalone prompt-only receptionist rail is allowed.
9. Human takeover, resume, assignment, and approval flows must reuse existing conversations and HITL patterns before new surfaces are added.
10. Org policy must fail closed for every action type with an explicit decision of `owner_only`, `approval_required`, or `agent_auto_allowed`.
11. Internal-only V1 closes before broad external CRM execution breadth; outward writes start with narrow contact/org/note sync only.
12. Sync `TASK_QUEUE.md`, `SESSION_PROMPTS.md`, `INDEX.md`, and `MASTER_PLAN.md` at every lane milestone.
