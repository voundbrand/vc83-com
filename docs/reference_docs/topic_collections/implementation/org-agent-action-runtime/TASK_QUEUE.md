## Queue rules

1. Allowed statuses are only: `READY`, `IN_PROGRESS`, `PENDING`, `BLOCKED`, `DONE`.
2. Only one row may be `IN_PROGRESS` globally unless an explicit lane exception is added here first.
3. Promote `PENDING` to `READY` only when every dependency is `DONE`.
4. Deterministic pick order is `P0` before `P1`, then lowest task ID.
5. Every row must run its `Verify` commands before moving to `DONE`.
6. Platform-owned CRM objects, links, activities, action items, and `objectActions` remain canonical; external CRM writes are downstream projections only.
7. Immutable activity and mutable pending work must stay separate: `org_agent_activity` never reopens, `org_agent_action_item` carries owner workflow state.
8. Veronica-style telephony must enter through the existing One-of-One Operator runtime and `agentSessions` flow; no standalone prompt-only receptionist rail is allowed.
9. Human takeover, resume, assignment, and approval flows must reuse existing conversations and HITL patterns before new surfaces are added.
10. Org policy must fail closed for every action type with an explicit decision of `owner_only`, `approval_required`, or `agent_auto_allowed`.
11. Internal-only V1 closes before broad external CRM execution breadth; outward writes start with narrow contact/org/note sync only.
12. Lane `E` owner workflow must support both list and Kanban views backed by the same canonical action-item state machine.
13. Agent filter chips for the Kanban view must cover the full active catalog taxonomy (default `all` plus per-agent filters from catalog metadata, not hardcoded UI-only labels).
14. Sync `TASK_QUEUE.md`, `SESSION_PROMPTS.md`, `INDEX.md`, and `MASTER_PLAN.md` at every lane milestone.
15. Every action-runtime row must declare whether it changes topology behavior (`single_agent_loop`, `pipeline_router`, `multi_agent_dag`, `evaluator_loop`) or remains kernel-stable.
16. Kernel-contract changes require versioning evidence and adapter-compatibility notes before promotion from `READY` to `IN_PROGRESS`.
17. All external awaits in delivery/provisioning/sync paths must be bounded by timeout + retry/fallback contracts before `DONE`.
18. Every runtime turn must persist a terminal state (`completed` or `failed`) exactly once, with receipt finalization proof.
19. New-agent onboarding must pass package-contract validation (prompt/tools/policy/memory/evals/rollout) before production enablement.
20. Promotion to canary must be blocked unless eval and SLO thresholds are attached to the row notes.
21. Every active agent in catalog scope must have a topology-profile inventory record before broad profile enforcement.
22. Any agent missing declared topology profile or package-compatibility evidence remains fail closed for production enablement.
23. Topology migration rows must record tool-calling pattern evidence and adapter/no-kernel-edit decisions per agent.
24. Architecture-preflight inventory (`OAR-033`) must be `DONE` before remaining lane `B` implementation rows (`OAR-005` onward) can move to `READY`.
